# 造園屋デイリー情報 ｜ 毎朝の発行手順書

毎朝5時（日本時間）の自動発行セッションは、この手順書に従って作業すること。
（2026-07-23 会社PCのClaudeが作成。自宅PC作成のNOTE_会社PCのClaudeへ.mdの内容を反映済み）

## 0. 前提

- 公開URL: https://helmetto2-prog.github.io/zoennews/ （GitHub Pages, mainブランチ配信）
- 最新号 = リポジトリ直下の `index.html`（毎回丸ごと差し替える）
- バックナンバー = `backnumber/YYYY-MM-DD.html` ＋ 一覧 `backnumber/index.html`
- 日付は**日本時間（JST = UTC+9）**で判断すること

## 1. ニュース収集（WebSearch）

以下の観点で検索し、**5〜7本**を選ぶ：

1. **地元優先**: 豊田市・愛知県・東海地方の造園/緑化/気象の話題
2. **業界ニュース**: 街路樹・公共緑化・行政動向・熱中症/安全衛生・病害虫（クビアカツヤカミキリ等）・補助金/助成金
3. **流行りの樹木・花**: 「M月D日の誕生花」を検索して1〜2本（造園目線の解説を添える）
4. **今日の学習メモ**: 豆知識を1本

**重複回避**: `backnumber/index.html` の直近5号ほどの見出しを確認し、同じ話題の繰り返しを避ける。続報なら「〜の続報」と明示する。

## 2. 紙面の作成（index.html）

- 既存の `backnumber/` 内の直近号の HTML をテンプレートとして使う（CSS・構造は変えない）
- `<title>` とヘッダーの日付を当日に更新（曜日も正しく）
- 構成: 業界ニュース（section.paper）→ 💪みどり造園の強みが活きるチャンス（div.chance）→ 流行りの樹木・花（section.cream）→ 今日の学習メモ（section.paper）→ 引用（div.pull）→ footer（生成: YYYY-MM-DD）
- 各記事: `<details>` 開閉式、`🌱 若手メモ`（新人向けの一言）、出典リンク必須
- 写真は転載しない（出典リンクで示す）
- **`</body>` 直前に必ずバックナンバーリンクを入れる**（消すと翌朝リンクが失われる）:
  `📚 バックナンバー一覧（全N号）` → `backnumber/` へのリンク（Nは当日号を含む通算号数）

## 3. バックナンバー登録（2ステップ）

1. 新しい `index.html` を `backnumber/YYYY-MM-DD.html` にコピー。
   ただし末尾のリンクは `📚 バックナンバー一覧へ戻る`（href="./"）に差し替える（既存号の末尾を参照）
2. `backnumber/index.html` を更新:
   - 一覧の**先頭**に新しい号のカード（div.issue）を追加（日付・曜日・記事数・見出しリスト・「この号を読む →」リンク。既存カードの構造をコピー）
   - ヘッダーの「全N号（YYYY年M月D日現在）」を +1 して更新

## 4. コミット＆プッシュ

- mainブランチに直接プッシュ（このリポジトリはそれが正: Pagesがmainを配信）
- コミットメッセージ例: 「2026年7月24日号を発行（自動）」
- push失敗時はネットワークエラーに限り 2s/4s/8s/16s の指数バックオフで最大4回リトライ

## 5. 完了報告でのURLの書き方（重要・ユーザー指摘 2026-07-23）

チャットでURLを伝えるとき、**全角カッコ（ ）や句読点でURLを囲まない**こと。
スマホ表示では閉じカッコや後ろの文字までリンクに含まれてしまい、壊れたアドレスに飛ぶ。

- ❌ 悪い例: `いつものアドレス（https://helmetto2-prog.github.io/zoennews/）に新しい号が…`
- ⭕ 良い例1: URLを単独の行に書く（前後に何も付けない）
- ⭕ 良い例2: Markdownの名前付きリンク `[今日の号を読む](https://helmetto2-prog.github.io/zoennews/)` を使う

## 6. してはいけないこと

- CSS・デザイン・カテゴリ構成の大幅変更（改善提案があればコミットせずユーザーへ報告）
- 過去号（backnumber内の既存ファイル）の書き換え・削除
- 未確認情報の断定的な記載（出典が取れない話題は載せない）
